# 发发 - 数据流转关系指南

## 版本：v4.0.0

## 文档目的

本指南详细介绍「发发 - 轻量级记账系统」的数据流转关系，帮助开发者理解：
- 核心系统与外挂功能的数据关系
- 基础数据与交易数据的关系
- 计算数据与存储数据的差异
- 完整的数据流向图解

---

## 一、系统架构总览

### 1.1 核心原则

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ★★★ 数据为核心，标签化存储，无损迭代 ★★★                      │
│                                                                 │
│   核心数据：transactions（交易是唯一事实来源）                   │
│   标签数据：members, merchants, tags, paymentChannels          │
│   衍生数据：balance, usedCredit, totalValue（使用时计算）        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 模块分类

| 分类 | 模块 | 说明 |
|------|------|------|
| **核心系统** | 交易记账 | 收支记录、转账、账户管理、分类管理、维度管理 |
| **外挂功能** | 信用卡管理 | 信用卡账单、还款、额度管理 |
| **外挂功能** | 贷款管理 | 贷款账户、还款计划、剩余金额 |
| **外挂功能** | 投资管理 | 投资账户、净值记录、收益统计 |
| **辅助功能** | 年度复盘 | 年度支出/收入统计、消费习惯分析 |
| **辅助功能** | 数据导入导出 | 数据迁移、备份恢复 |

```
│  │  Layer 2: 基础定义层（业务对象定义）                                  │   │
│  │                                                                     │   │

│                                    │                                        │

│  │                                                                     │   │

│                                                                             │
```

```
│  │   │  3. 外挂数据依赖交易计算（如已用额度 = SUM(消费) - SUM(还款)）  │    │ │
│  │   │  4. 可独立使用，不影响核心记账功能                             │    │ │
│  │   │  5. 关闭外挂功能时，仅删除定义表，已交易数据保留为普通交易      │    │ │
│  │   └─────────────────────────────────────────────────────────────┘    │ │
│  │                                                                       │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
